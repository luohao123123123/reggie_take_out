1.安装依赖包：yum -y install gcc pcre-devel zlib-devel openssl openssl-devel
2.下载nginx安装包： wget https://nginx.org/download/nginx-1.16.1.tar.gz
3.解压
4.进入nginx目录
5.指定安装目录（目录需要提前创建）：./configure --prefix=/opt/apps/nginx
6.执行安装：make install

        nginx的作用
1.部署静态资源
2.反向代理
3.负载均衡

        为什么使用nginx部署
相对于tomcat，nginx处理静态资源的能力更加高效，所以在生产环境下，一般都会将静态资源部署到nginx中。部署非常简单，只要将文件复制到nginx的html目录下即可

        反向代理
1.什么是正向代理
    即用户和原始服务器之间的服务器
2.什么是反向代理
    即用户和目标服务器之间的服务器，对于用户而言，反向代理服务器就相对于目标服务器
3.区别
    1、正向代理其实是客户端的代理，帮助客户端访问其无法访问的服务器资源。反向代理则是服务器的代理，帮助服务器做负载均衡，安全防护等。
    2、正向代理和反向代理的作用和目的不同。正向代理主要是用来解决访问限制问题。而反向代理则是提供负载均衡、安全防护等作用。二者均能提高访问速度。
4.使用nginx，在nginx.conf中配置，就能使用反向代理，配置192.168.213.60为192.168.213.61的反向代理服务器，那么就能通过192.168.213.60访问192.168.213.61的服务
